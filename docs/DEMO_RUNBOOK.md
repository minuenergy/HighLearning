# SocraTeach Demo Runbook

Last updated: 2026-04-12

이 문서는 발표 직전에 바로 펼쳐보고 따라갈 수 있는 시연용 대본입니다.
목표는 `짧고 안정적으로`, 그리고 `교사 문제 -> 제품 해결 -> 학생 학습 -> 다시 교사 인사이트` 흐름이 한 번에 보이게 하는 것입니다.

## 1. 데모 한 줄 소개

SocraTeach는 교사가 쓰는 교재나 자료를 바탕으로 시험을 만들고 배포한 뒤, 학생의 오답 복기 대화와 시험 결과를 다시 교사에게 돌려줘 보강수업까지 연결하는 AI 학습 운영 도구입니다.

## 2. 데모 계정

어드민:
- `admin@socrateach.school`
- `SocraTeachAdmin!2026`

교사:
- `teacher01@socrateach.school`
- `SocraTeachTeacher!2026`

학생:
- `student001@socrateach.school`
- `SocraTeachStudent!2026`

## 3. 발표 전 3분 체크

순서:
1. `cd socrateach && npm run qa:final`
2. 결과가 `총 4개 시나리오 통과`인지 확인
3. 브라우저 탭 3개 준비
교사용 1개, 학생용 1개, 예비용 1개
4. 로그인 정보 복사해두기
5. 발표에서는 `teacher01`, `student001`만 사용하기

## 4. 5분 데모 버전

### 4-1. 흐름 요약

1. 교사 홈
2. 학생 관리
3. 시험지 제작
4. 학생 시험 응시
5. 다시 교사 분석

### 4-2. 대본

#### 0:00-0:30 문제 정의

말할 내용:
- 교사는 시험을 만드는 데 시간이 많이 들고,
- 학생이 왜 틀렸는지는 시험 점수만으로는 잘 보이지 않습니다.
- 저희는 시험 제작, 학생 복기, 교사 보강 인사이트를 하나의 흐름으로 묶었습니다.

#### 0:30-1:20 교사 홈

계정:
- `teacher01@socrateach.school`

클릭:
1. 로그인
2. `Teacher Home` 진입
3. 운영 카드와 AI 대화 브리핑 확인

말할 내용:
- 여기서는 반별 운영 현황과 숙제/시험 흐름을 한눈에 봅니다.
- 그리고 오른쪽의 AI 브리핑은 학생들의 튜터 대화를 읽고 공통 오개념과 교사용 말하기 포인트를 정리해줍니다.

강조 포인트:
- 단순 점수판이 아니라 `수업 운영 보드`

#### 1:20-2:00 학생 관리

클릭:
1. 좌측 사이드바 `학생 관리`
2. 반 선택
3. 학생 한 명 선택

말할 내용:
- 학생 관리에서는 단순 성적뿐 아니라,
- 최근 튜터 대화 기반으로 어떤 개념을 반복해서 헷갈리는지 바로 봅니다.
- 위의 `학생 개별 Gemini 브리핑`은 교사가 지금 이 학생에게 어떤 개입을 해야 하는지 짧게 정리한 카드입니다.

강조 포인트:
- 학생별 메모
- 취약 개념
- 질문 패턴

#### 2:00-2:50 시험지 제작

클릭:
1. 좌측 사이드바 `시험지 제작`
2. 초안 하나 선택
3. 워크스페이스 열기
4. 문항 하나만 보여주기
5. 예약/알림 운영 상태 카드 보여주기

말할 내용:
- 교사는 교재 초안이나 업로드 자료 기반 draft를 바로 열 수 있습니다.
- 문항을 수정하고 자동 저장한 뒤, 학생에게 예약 배포할 수 있습니다.
- 아래 카드에서는 지금 예약된 시험, 진행 중 시험, 미제출 알림 상태까지 같이 운영합니다.

강조 포인트:
- 시험 제작과 배포가 분리되지 않고 한 흐름

#### 2:50-3:50 학생 시험 응시

계정 전환:
- `student001@socrateach.school`

클릭:
1. 학생 홈
2. 알림 카드 확인
3. 시험 페이지 진입
4. 시험 제출
5. 오답 복기 버튼 또는 튜터 이동

말할 내용:
- 학생은 로그인하자마자 해야 할 시험 알림을 봅니다.
- 시험을 제출하면 자동 채점이 되고,
- 틀린 문제는 바로 정답을 주지 않고 소크라테스식 복기로 이어집니다.

강조 포인트:
- `정답 제공`이 아니라 `스스로 설명하게 만드는 복기`

#### 3:50-5:00 다시 교사 화면

계정 전환:
- 다시 `teacher01`

클릭:
1. `학생 관리` 또는 `과목 관리`
2. 브리핑 카드 확인

말할 내용:
- 학생의 복기 대화와 시험 결과는 다시 교사에게 돌아옵니다.
- 그래서 교사는 누가 무엇을 왜 틀렸는지, 다음 수업에서 뭘 먼저 설명해야 할지 바로 알 수 있습니다.

마무리 한 줄:
- 저희는 시험을 만드는 AI가 아니라, `교사의 수업 운영 전체를 보조하는 AI`를 만들었습니다.

## 5. 10분 데모 버전

### 5-1. 흐름 요약

1. 어드민 초대코드
2. 교사 홈
3. 학생 관리
4. 과목 관리
5. 시험지 제작
6. 학생 응시와 오답 복기
7. 다시 교사 인사이트

### 5-2. 대본

#### 0:00-1:00 오프닝

말할 내용:
- 교사는 시험을 만들고, 배포하고, 채점 결과를 보고, 보강수업까지 준비해야 합니다.
- 하지만 이 과정이 다 끊어져 있어서 학생이 왜 틀렸는지 추적하기 어렵습니다.
- SocraTeach는 이 흐름을 하나로 연결합니다.

#### 1:00-2:00 어드민 흐름

계정:
- `admin@socrateach.school`

클릭:
1. 로그인
2. `승인 관리`
3. 교사 초대코드 확인 또는 발급

말할 내용:
- 학교에서는 최초 어드민 교사가 전체 교사 온보딩을 관리할 수 있습니다.
- 교사는 어드민이 발급한 초대코드로 들어오고, 학생은 다시 담당 교사의 반 초대코드로 들어옵니다.

#### 2:00-3:30 교사 홈

계정:
- `teacher01@socrateach.school`

클릭:
1. 교사 홈
2. 운영 카드
3. 달력형 보드
4. AI 대화 브리핑

말할 내용:
- 홈 화면은 단순 메뉴가 아니라 운영 대시보드입니다.
- 어떤 반이 밀리고 있는지, 어떤 오개념이 반복되는지, 교사가 수업에서 어떻게 말하면 좋을지를 바로 봅니다.

#### 3:30-5:00 학생 관리

클릭:
1. `학생 관리`
2. `중1 1반`
3. 학생 선택
4. 메모, 학생 개별 Gemini 브리핑, 최근 대화 확인

말할 내용:
- 교사는 학생별로 메모를 남길 수 있고,
- AI는 최근 튜터 대화를 바탕으로 취약 개념, 반복 오개념, 질문 패턴을 요약합니다.
- 즉, 단순 성적표가 아니라 `이 학생을 다음 수업에서 어떻게 도와야 하는지`까지 연결됩니다.

#### 5:00-6:20 과목 관리

클릭:
1. `과목 관리`
2. 국어 선택
3. 과목별 Gemini 브리핑 확인

말할 내용:
- 과목 관리에서는 반별 편차와 가장 어려운 문항, 공통 혼동 패턴을 봅니다.
- 과목 브리핑은 지금 어떤 단원을 먼저 보강해야 하는지 우선순위를 짧게 정리합니다.

#### 6:20-7:40 시험지 제작

클릭:
1. `시험지 제작`
2. 왼쪽 4단계 선택
3. 워크스페이스 열기
4. 문항/근거 조각/학생용 미리보기 확인
5. 예약 배포 상태 카드 확인

말할 내용:
- 시험은 교재 또는 업로드 자료에서 출발합니다.
- 생성된 문항은 교사가 바로 수정할 수 있고,
- 각 문항은 근거 페이지와 텍스트 조각을 같이 확인할 수 있습니다.
- 그리고 배포 후에는 예약/알림 상태를 같은 화면에서 운영합니다.

#### 7:40-8:50 학생 응시와 복기

계정:
- `student001@socrateach.school`

클릭:
1. 학생 홈
2. 시험 알림 카드
3. 시험 응시
4. 제출
5. 오답 복기
6. 해설/근거 조각 확인

말할 내용:
- 학생은 해야 할 시험을 바로 확인하고,
- 제출하면 자동 채점 후 오답 복기로 넘어갑니다.
- 정답을 바로 보여주지 않고, 질문을 통해 학생이 스스로 설명하도록 유도합니다.

#### 8:50-10:00 교사로 복귀

계정:
- 다시 `teacher01`

클릭:
1. 학생 관리 또는 과목 관리
2. 브리핑 카드 다시 확인

말할 내용:
- 학생이 시험을 보고 복기한 정보는 다시 교사 인사이트로 돌아옵니다.
- 그래서 교사는 다음 보강수업에서 무엇을 말하고, 어떤 학생을 먼저 봐야 할지 바로 알 수 있습니다.

클로징:
- SocraTeach는 시험 제작 AI가 아니라,
- `교사의 가르침을 중심에 두고 시험, 복기, 분석을 연결하는 운영 도구`입니다.

## 6. 발표자가 꼭 강조하면 좋은 문장

- 시험을 만드는 것에서 끝나지 않고, 결과가 다시 교사에게 돌아옵니다.
- 학생은 정답을 바로 받지 않고, 스스로 설명하면서 복기합니다.
- 교사는 점수만 보는 것이 아니라, 왜 틀렸는지와 어떻게 다시 설명할지를 함께 봅니다.
- AI가 교사를 대체하는 게 아니라, 교사의 보강수업 준비 시간을 줄여줍니다.

## 7. 발표 중 피하면 좋은 것

- 처음부터 자료 업로드 전체 파이프라인을 길게 보여주기
- 너무 많은 계정을 왔다 갔다 하기
- 시험지 제작에서 문항을 여러 개 길게 수정하기
- AI 응답을 실시간으로 오래 기다리게 두기

권장:
- 한 화면에서 핵심 카드만 짧게 보여주고 다음 화면으로 넘어가기

## 8. 장애 발생 시 대체 동선

### 상황 1. AI 응답이 느릴 때

대체:
- 이미 렌더된 Gemini 브리핑 카드 위주로 설명
- “이 브리핑은 실제 대화 원문과 시험 데이터를 바탕으로 생성됩니다”라고 설명하고 다음 화면 이동

### 상황 2. 시험 배포를 다시 하기 애매할 때

대체:
- 이미 생성된 시험 카드와 예약/알림 운영 상태 카드 위주로 설명
- 학생 화면에서는 기존 시험 목록과 복기된 결과를 보여주기

### 상황 3. 학생 전환이 느릴 때

대체:
- 학생 관리 화면의 최근 대화, 학생 브리핑, 과목 관리 브리핑으로 마무리

## 9. 발표 직전 최종 체크

1. `npm run qa:final` 통과
2. 어드민, 교사, 학생 계정 로그인 정보 복사
3. 브라우저 탭 2개 이상 준비
4. 교사 계정은 `teacher01`, 학생 계정은 `student001`만 사용
5. 발표는 `교사 문제 -> 학생 경험 -> 다시 교사 인사이트` 순서 유지
