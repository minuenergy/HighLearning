# SocraTeach Presentation Slides

Last updated: 2026-04-12

이 문서는 실제 발표 슬라이드에 바로 붙여 넣을 수 있는 제목, 본문, 발표 멘트를 정리한 문서입니다.
기본 기준은 `7장 구성`, `5~10분 발표`, `교사 문제 -> 제품 해결 -> 학생 경험 -> 다시 교사 인사이트` 흐름입니다.

## 1. 전체 구성

1. 문제 정의
2. 해결 방식
3. 교사 홈
4. 학생 관리 / 과목 관리
5. 시험지 제작
6. 학생 시험 응시와 오답 복기
7. 결론

## 2. 슬라이드별 문안

### 슬라이드 1. 문제 정의

슬라이드 제목:
- 교사는 시험 이후를 더 오래 준비합니다

슬라이드 부제:
- 시험 제작, 오답 분석, 보강수업 준비가 모두 분리돼 있습니다

슬라이드 본문:
- 시험 제작과 배포에 시간이 많이 듭니다
- 점수만으로는 학생이 왜 틀렸는지 알기 어렵습니다
- 학생이 AI를 쓰면 정답만 바로 얻어버릴 위험이 있습니다

발표 멘트:
- 학교 현장에서 교사는 시험을 만들고 끝나는 것이 아니라, 그 이후의 해석과 보강까지 책임집니다.
- 그런데 시험 제작, 학생 오답 분석, 보강수업 준비가 각각 따로 놀기 때문에 시간이 많이 들고 흐름이 끊깁니다.
- 저희는 바로 이 단절을 줄이는 데서 출발했습니다.

화면/디자인 메모:
- 왼쪽에는 교사 업무 흐름 아이콘
- 오른쪽에는 분절된 프로세스 화살표

### 슬라이드 2. 해결 방식

슬라이드 제목:
- SocraTeach는 시험 제작부터 복기와 보강까지 연결합니다

슬라이드 부제:
- 시험 생성 AI가 아니라, 교사 운영 전체를 돕는 AI입니다

슬라이드 본문:
- 교재나 수업 자료에서 시험 초안을 생성합니다
- 학생은 정답 대신 소크라테스식 질문으로 오답을 복기합니다
- 결과는 다시 교사에게 돌아와 공통 오개념과 보강 포인트가 됩니다

발표 멘트:
- SocraTeach는 교재 기반 시험 생성에서 멈추지 않습니다.
- 학생은 시험을 제출한 뒤 정답을 바로 받는 대신, 질문 중심 복기를 통해 스스로 설명하게 됩니다.
- 그리고 그 대화와 시험 결과가 다시 교사 화면으로 돌아와 다음 수업 준비에 쓰입니다.

화면/디자인 메모:
- `교재 -> 시험 -> 학생 복기 -> 교사 인사이트` 순환 다이어그램

### 슬라이드 3. 교사 홈

슬라이드 제목:
- 교사는 운영 상태와 공통 오개념을 한눈에 봅니다

슬라이드 부제:
- 반 운영 현황, 일정, AI 브리핑을 한 화면에 모았습니다

슬라이드 본문:
- 반별 운영 카드와 달력형 진도 보드
- 학생 튜터 대화 기반 AI 브리핑
- 공통 오개념, 질문 패턴, 수업용 talk track 제공

발표 멘트:
- 교사 홈은 단순 메뉴 화면이 아니라 운영 대시보드입니다.
- 어떤 반이 밀리고 있는지, 어떤 개념을 많이 틀리는지, 학생들이 어떤 방식으로 질문하는지를 함께 봅니다.
- 특히 AI 브리핑은 교사가 다음 수업에서 바로 사용할 수 있는 설명 포인트까지 짧게 정리해줍니다.

화면/디자인 메모:
- 실제 teacher dashboard 스크린샷 사용

### 슬라이드 4. 학생 관리 / 과목 관리

슬라이드 제목:
- 학생 단위와 과목 단위 모두에서 이해 상태를 해석합니다

슬라이드 부제:
- 성적표를 넘어서, 왜 틀렸는지까지 보이게 합니다

슬라이드 본문:
- 학생별 Gemini 브리핑과 교사 메모
- 반복 오개념, 질문 패턴, 최근 튜터 대화 요약
- 과목별 평균 성적, 어려운 문항, 과목 브리핑

발표 멘트:
- 학생 관리에서는 한 학생이 최근 어떤 개념을 반복해서 헷갈렸는지 바로 확인할 수 있습니다.
- 과목 관리에서는 반별 편차와 가장 어려운 문항, 공통 혼동 패턴을 확인할 수 있습니다.
- 즉, 교사는 점수뿐 아니라 왜 어려워했는지와 어디부터 설명해야 하는지를 함께 보게 됩니다.

화면/디자인 메모:
- 왼쪽 학생 관리, 오른쪽 과목 관리 화면을 분할 배치

### 슬라이드 5. 시험지 제작

슬라이드 제목:
- 교사는 초안을 수정하고 예약 배포할 수 있습니다

슬라이드 부제:
- 교재 근거 확인, 문항 편집, 학생용 미리보기를 한 워크스페이스에서 처리합니다

슬라이드 본문:
- 교재 초안과 업로드 자료 draft를 바로 선택
- 문항 편집, autosave, 학생용 미리보기 지원
- 예약 배포와 알림 운영 상태까지 같은 화면에서 관리

발표 멘트:
- 시험은 교재나 업로드 자료에서 생성된 초안으로 시작합니다.
- 교사는 문항을 바로 수정하고, 근거 페이지와 텍스트 조각을 확인하며, 학생에게 보일 형태도 미리 볼 수 있습니다.
- 이후 예약 시간과 마감일을 설정해 학생에게 배포하고, 운영 상태까지 이어서 확인합니다.

화면/디자인 메모:
- 시험지 제작 워크스페이스 전체 화면 사용

### 슬라이드 6. 학생 시험 응시와 오답 복기

슬라이드 제목:
- 학생은 정답을 바로 받지 않고, 스스로 설명하며 복기합니다

슬라이드 부제:
- 시험 응시와 AI 튜터가 분리되지 않고 자연스럽게 이어집니다

슬라이드 본문:
- 학생 홈에서 시험 알림과 마감 정보를 바로 확인
- 제출 후 자동 채점
- 오답은 소크라테스식 질문으로 복기한 뒤 해설과 근거를 확인

발표 멘트:
- 학생은 로그인하자마자 해야 할 시험을 보고 바로 응시할 수 있습니다.
- 제출 후 자동 채점이 되지만, 틀린 문제는 정답을 바로 열어주지 않습니다.
- 대신 질문 중심 복기를 통해 학생이 스스로 설명해 보게 하고, 그 이후에 해설과 근거를 확인하게 합니다.

화면/디자인 메모:
- 학생 시험 화면과 튜터 화면을 순서대로 배치

### 슬라이드 7. 결론

슬라이드 제목:
- SocraTeach는 교사의 가르침을 더 잘 이어주는 도구입니다

슬라이드 부제:
- 시험 생성, 학생 복기, 교사 보강이 하나의 흐름으로 연결됩니다

슬라이드 본문:
- 교사 중심의 학습 운영 도구
- 정답 제공보다 복기와 이해를 우선
- 학생 데이터가 다시 교사의 수업 준비로 연결

발표 멘트:
- 저희가 만들고 싶은 것은 문제를 자동으로 만들어주는 도구 하나가 아닙니다.
- 시험, 복기, 보강수업 준비까지 하나의 흐름으로 묶어 교사의 실제 가르침을 돕는 운영 도구입니다.
- SocraTeach는 AI가 답을 대신하는 것이 아니라, 교사와 학생이 더 잘 배우고 가르치도록 연결하는 제품입니다.

화면/디자인 메모:
- 핵심 메시지 3개를 크게 배치

### 슬라이드 2
- 제목: SocraTeach는 이 흐름을 하나로 연결합니다
- 본문: 시험 생성 -> 학생 복기 -> 교사 인사이트

### 슬라이드 3
- 제목: 교사는 운영 현황과 공통 오개념을 한눈에 봅니다
- 본문: 운영 카드, 브리핑, 위험 지도

### 슬라이드 4
- 제목: 학생별/과목별로 왜 틀렸는지까지 해석합니다
- 본문: 학생 브리핑, 과목 브리핑, 반복 오개념

### 슬라이드 5
- 제목: 시험은 교재 기반 초안에서 시작해 예약 배포까지 이어집니다

### 슬라이드 6
- 제목: 학생은 정답 대신 복기를 시작합니다
- 본문: 자동 채점, 오답 복기, 해설 공개

### 슬라이드 7
- 제목: SocraTeach는 교사의 수업 운영 전체를 돕는 AI입니다
- 본문: 생성 AI를 넘어, 실제 가르침을 지원

## 4. 발표자가 바로 꺼내 쓸 핵심 문장

- 저희는 시험 생성 AI가 아니라, 교사의 수업 운영 전체를 보조하는 AI를 만들고 있습니다.
- 학생이 AI에게 답을 바로 받는 것이 아니라, 질문을 통해 스스로 설명하게 만드는 흐름을 중요하게 봤습니다.
- 학생의 시험 결과와 튜터 대화가 다시 교사에게 돌아오기 때문에, 다음 보강수업 준비가 훨씬 빨라집니다.
- 점수만 보여주는 것이 아니라 왜 틀렸는지와 어떻게 다시 설명할지를 함께 보여주는 것이 핵심입니다.
